# Проектирование реляционной базы данных

## Назначение реляционной базы данных

Реляционная база данных в системе используется для хранения пользовательских данных и результатов взаимодействия с системой.

В отличие от графовой базы знаний, которая содержит учебный контент (сценарии, шаблоны, лексику), реляционная база данных отвечает за:

- идентификацию пользователей;
- хранение истории прохождения тренировок;
- фиксацию результатов выполнения заданий;
- анализ прогресса пользователя.

Таким образом, реляционная модель реализует эксплуатационный уровень системы, связанный с пользовательским опытом.

---

## Основные сущности

В рамках проектирования выделены три ключевые сущности:

1. Пользователь (User)  
2. Тренировочная сессия (TrainingSession)  
3. Ответ пользователя (UserAnswer)  

Данный набор является достаточным для текущего этапа проектирования и покрывает все задачи, возложенные на реляционную базу данных.

---

## Сущность «Пользователь»

Сущность «Пользователь» предназначена для идентификации субъекта взаимодействия с системой.

### Назначение:
- хранение информации о пользователе;
- разграничение данных между пользователями;
- накопление индивидуальной истории обучения.

### Основные атрибуты:
- `id` — уникальный идентификатор пользователя;
- `name` — имя пользователя (опционально);
- `created_at` — дата регистрации.

---

## Сущность «Тренировочная сессия»

Сущность «Тренировочная сессия» фиксирует отдельный завершённый акт прохождения тренировки.

### Назначение:
- агрегирование результатов пользователя;
- привязка результатов к конкретному сценарию;
- фиксация временных характеристик.

### Основные атрибуты:
- `id` — идентификатор сессии;
- `user_id` — ссылка на пользователя;
- `scenario_id` — идентификатор сценария (из базы знаний);
- `started_at` — время начала сессии;
- `finished_at` — время завершения;
- `correct_answers_count` — количество правильных ответов;
- `total_questions` — общее количество заданий.

### Роль в системе:
Сессия является верхним уровнем агрегации и позволяет рассматривать тренировку как единое событие.

---

## Сущность «Ответ пользователя»

Сущность «Ответ пользователя» хранит детализированные данные о выполнении заданий.

### Назначение:
- фиксация ответов на отдельные задания;
- определение корректности выполнения;
- анализ структуры ошибок пользователя.

### Основные атрибуты:
- `id` — идентификатор ответа;
- `session_id` — ссылка на тренировочную сессию;
- `question_id` — идентификатор задания (из базы знаний);
- `user_answer` — ответ пользователя;
- `correct_answer` — правильный ответ;
- `is_correct` — признак корректности ответа.

---

## Связи между сущностями

Структура связей:

- Пользователь (1) → (N) Тренировочная сессия  
- Тренировочная сессия (1) → (N) Ответ пользователя  

Таким образом, модель имеет иерархическую структуру:

Пользователь
  └── Тренировочные сессии
         └── Ответы пользователя
